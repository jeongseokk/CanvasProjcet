main_android.dart 쉬운 해설

대상 파일: project_canvas/lib/main_android.dart
목표: Flutter를 처음 보는 사람도 파일을 위에서 아래로 따라가며 “이 줄이 왜 있는지” 이해할 수 있게 설명

이 파일의 정체 이 파일은 단순한 화면 파일이 아닙니다. 앱 실행, 메인 화면, 캔버스 그리기, 배경 이미지 업로드, 좌표 변환, Z 값 관리, TCP 전송, 웹뷰까지 거의 전부 들어 있습니다.

한 줄 요약 이미지 위에 경로를 그리고, 그 경로를 점 좌표로 바꾼 다음, mm 단위와 Z 값을 붙여 JSON으로 서버에 보내는 도구입니다.

dart:convert: JSON 인코딩에 사용합니다. 나중에 좌표 데이터를 jsonEncode로 보내기 위해 필요합니다.

dart:io: 파일, 소켓, 네트워크 인터페이스, 플랫폼 구분 같은 OS 기능을 쓰기 위해 필요합니다.

dart:math as math: 원 둘레 계산, 최소/최대 계산, 삼각함수 등에 씁니다.

flutter/gestures: 마우스 버튼 종류 같은 pointer 입력 처리에 씁니다.

image_picker: 갤러리에서 이미지를 고를 때 씁니다.

material.dart: Flutter UI의 기본 재료입니다.

services.dart: 키보드 상태, 예를 들면 Shift 키가 눌렸는지 확인할 때 씁니다.

webview_flutter, webview_windows: 웹 페이지를 앱 안에서 보여줄 때 씁니다.

foundation.dart: 현재 플랫폼이 Android, iOS, Windows인지 구분할 때 씁니다.

url_launcher: 브라우저를 외부로 여는 데 씁니다.

4. CanvasHomePage와 createState

StatefulWidget: 화면이 계속 바뀌는 위젯입니다. 이 앱은 그리고, 선택하고, 보내고, 상태 문구가 계속 바뀌므로 StatefulWidget이 맞습니다.

6. 네트워크 보조 함수

Future<String> _networkDebugInfo() async { ... }
List<int>? _tryParseIpv4(String value) { ... }
bool _same24Subnet(String a, String b) { ... }
Future<InternetAddress?> _selectSourceAddress(String host) async { ... }

_networkDebugInfo(): 현재 장치의 네트워크 인터페이스 이름과 IP를 문자열로 모읍니다. 연결 실패 시 진단 정보로 씁니다.

_tryParseIpv4(): 192.168.0.10 같은 문자열을 [192, 168, 0, 10] 배열로 바꿉니다.

_same24Subnet(): 두 IP의 앞 세 구간이 같은지 비교해 같은 대역인지 확인합니다.

_selectSourceAddress(): 장치에 IP가 여러 개 있을 때, 목표 host와 같은 대역의 로컬 IP를 우선 선택합니다.

이 블록의 목적은 “그냥 연결”만 하는 것이 아니라, 연결 실패 시 원인을 더 잘 보이고, 여러 네트워크 인터페이스가 있을 때 조금 더 안정적으로 연결하려는 것입니다.

7. initState와 dispose

@override
void initState() {
  super.initState();
  _bgWidthController.text = '594.0';
  _bgHeightController.text = '420.0';
  _userFrameMmSize = const Size(594.0, 420.0);
  _userFramePxSize = const Size(594.0, 420.0);
  _mmPerPx = 1.0;
  _mmPerPxController.text = _mmPerPx.toStringAsFixed(3);
}

initState(): 이 화면이 처음 생길 때 한 번 실행됩니다.

기본 프레임을 A2 크기인 594 x 420으로 잡아 입력창과 내부 상태를 같이 초기화합니다.

_mmPerPx = 1.0: 처음에는 1픽셀을 1mm처럼 다루는 단순 기준으로 시작합니다.

@override
void dispose() {
  _hostController.dispose();
  _portController.dispose();
  _zController.dispose();
  _stepPxController.dispose();
  _mmPerPxController.dispose();
  _bgWidthController.dispose();
  _bgHeightController.dispose();
  _socket?.destroy();
  super.dispose();
}

dispose(): 이 화면이 사라질 때 마지막으로 정리하는 함수입니다.

TextEditingController는 내부 상태와 리스너를 들고 있으므로 화면이 끝날 때 해제해야 합니다.

_socket?.destroy(): 열려 있는 TCP 연결이 있다면 종료합니다.

super.dispose(): 부모 State의 정리 작업도 실행합니다.

8. 사용자가 입력한 설정 적용 함수

void _applyMmPerPx() { ... }
void _applyStepPx() { ... }
void _applyZToAll() { ... }
void _applyBackgroundSize() { ... }
void _setPresetSize(double widthMm, double heightMm) { ... }

_applyMmPerPx(): 사용자가 입력한 mm/px 값을 읽고 내부 상태에 적용합니다. 잘못된 값이면 status에 오류를 표시합니다.

_applyStepPx(): 점 간격을 바꾸고, 현재 그리고 있던 도형이 있으면 즉시 다시 계산합니다.

_applyZToAll(): 현재 선택된 stroke 또는 전체 stroke의 모든 점에 같은 Z 값을 넣습니다.

_applyBackgroundSize(): 사용자가 입력한 배경 실제 크기(mm)를 프레임 기준으로 적용합니다.

_setPresetSize(): A4, A2, B4 같은 프리셋 버튼을 눌렀을 때 호출됩니다.

9. 포인터 입력: 그리기, 이동, 선택, 점 편집

void _onPointerDown(PointerDownEvent event) { ... }
void _onPointerMove(PointerMoveEvent event) { ... }
void _onPointerUp(PointerUpEvent event) { ... }

_onPointerDown

  • 우클릭이면 stroke 이동 시작
  • Shift + pen 모드면 점 편집 시작
  • Select 모드면 선택 박스 시작
  • 그 외에는 새 도형 그리기 시작

_onPointerMove

  • 이동 중이면 stroke 전체 이동
  • 점 편집 중이면 선택한 점 위치 수정
  • 선택 중이면 사각형 갱신
  • 그리기 중이면 현재 도형 갱신 후 점 다시 계산

_onPointerUp

  • 이동 중이었으면 이동 종료
  • 점 편집 중이었으면 편집 종료
  • 선택 중이었으면 선택 박스로 stroke 선택
  • 그리기 중이었으면 점을 확정하고 stroke 목록에 저장

이 세 함수가 사용자 입력의 중심입니다. 이 앱은 버튼보다 캔버스 상호작용이 중요하므로 이 블록이 매우 핵심입니다.

10. 현재 상태 지우기와 삭제

void _reset() { ... }
void _clearCurrentDrawing() { ... }
void _deleteSelectedOrLast() { ... }

_reset(): 전체를 초기화합니다. 그린 것, 선택 상태, 임시 점, stroke 목록을 거의 모두 비웁니다.

_clearCurrentDrawing(): 지금 그리고 있던 임시 도형만 지웁니다.

_deleteSelectedOrLast(): 선택된 stroke가 있으면 그 stroke를 지우고, 없으면 마지막 stroke를 지웁니다.

12. TCP 연결 만들기

Future<void> _ensureConnected() async { ... }

Host와 Port 입력값이 유효한지 먼저 검사합니다.

이미 같은 host, port에 연결된 소켓이 있으면 재사용합니다.

다른 곳에 연결 중이면 기존 소켓을 닫고 새 연결을 만듭니다.

Socket.connect(...)로 실제 TCP 연결을 시도합니다.

socket.listen(...)으로 서버가 보내는 응답도 받아 status에 반영합니다.

에러가 나면 네트워크 인터페이스 정보까지 함께 상태 메시지에 적어 디버깅을 돕습니다.

13. 전송: JSON 만들고 보내기

Future<void> _sendJson() async { ... }
Map<String, dynamic> _buildPayload() { ... }
int _payloadPointCount() { ... }

_sendJson(): 현재 그린 도형 또는 stroke가 있는지 검사한 뒤, 연결을 확보하고 JSON을 전송합니다.

_buildPayload(): 실제 전송할 데이터 구조를 만듭니다.

_payloadPointCount(): 몇 개의 포인트가 전송되는지 UI 표시용으로 계산합니다.

전송 형식

{
  "points": [
    [12.34, 56.78, 0.00],
    [14.11, 58.40, 0.00],
    ["000"]
  ]
}

[x, y, z] 하나가 점 하나입니다. stroke가 여러 개일 때는 stroke 끝마다 ["000"] 마커를 추가해 구분합니다.

14. 가장 중요한 함수: _rebuildPoints

void _rebuildPoints() { ... }

이 함수는 “사용자가 그린 모양”을 “전송 가능한 점들”로 바꿉니다.

모드에 따라 다르게 처리합니다.

  • line: 시작점과 끝점 사이를 일정 간격으로 나눔
  • circle: 원 둘레를 일정 간격으로 샘플링
  • rect: 사각형 테두리를 따라 샘플링
  • pen: 사용자가 그린 자유곡선을 다시 일정 간격으로 재샘플링

그 뒤 _pointsImage, _pointsPx, _pointsMm, _deltasMm를 모두 갱신합니다.

즉 이 함수가 있어야 “보이는 선”이 “실제 데이터”로 바뀝니다.

15. 좌표 계산과 샘플링 수학 함수

_applyTransform(): 이미지 좌표에 scale을 적용해 mm 좌표로 바꿉니다. 필요하면 y 뒤집기나 회전도 넣을 수 있게 구조가 잡혀 있습니다.

_buildDeltas(): 각 점과 다음 점 사이의 차이 벡터를 계산합니다.

21. 좌표계 변환: 화면 좌표와 이미지 좌표를 오가기

29. 이 파일 전체를 다시 한 번 흐름으로 요약

  1. 앱이 시작되면 CanvasApp이 실행되고 CanvasHomePage가 첫 화면으로 열린다.
  2. 사용자는 배경 이미지를 올리거나 프레임 크기를 설정한다.
  3. 캔버스에서 line, pen, circle, rect 모드로 경로를 그린다.
  4. 앱은 그 경로를 일정 간격 점들로 재계산한다.
  5. 화면 좌표를 이미지 좌표로 바꾸고, 다시 mm 좌표로 변환한다.
  6. 점마다 Z 값을 붙인다.
  7. 필요하면 stroke를 선택, 이동, 스무딩, 점 편집한다.
  8. Send 버튼을 누르면 JSON payload를 만들고 TCP로 서버에 전송한다.

30. 초보자 관점에서 꼭 기억할 핵심 10개

  • CanvasApp은 앱 껍데기, CanvasHomePage는 실제 작업 화면입니다.
  • StatefulWidget이라서 화면 상태가 계속 변할 수 있습니다.
  • _가 붙은 변수와 함수는 이 파일 내부용입니다.
  • Offset은 좌표 한 점입니다.
  • Stroke는 한 번 그린 선 묶음입니다.
  • _rebuildPoints()가 “그림”을 “전송할 점 데이터”로 바꾸는 핵심 함수입니다.
  • _canvasToImageClamped()와 _imageToCanvas()가 좌표계를 연결합니다.
  • _ensureConnected()가 소켓 연결을 관리합니다.
  • _buildPayload()가 최종 JSON을 만듭니다.
  • LinePainter는 데이터를 실제 화면에 그리는 전용 클래스입니다.
